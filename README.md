## 🎯 코스 생성 알고리즘

```
Step 1: 장소 수집
   ├─ 한국관광공사 API (공식 인증 장소)
   └─ 카카오맵 검색 (다양한 선택지)
   
Step 2: 거리 필터링
   └─ 활동 시간대별 거리 범위 적용
   
Step 3: 카테고리 분류
   ├─ 정규식 패턴 매칭
   ├─ 9가지 카테고리로 자동 분류
   └─ Fallback: 기타 카테고리
   
Step 4: 활동별 조합 규칙 생성
   ├─ 활동에 맞는 2개 카테고리 조합
   └─ 최대 15가지 조합 템플릿 생성
   
Step 5: 코스 구축
   ├─ 각 조합으로 실제 코스 생성
   ├─ 거리 검증 (10km 이내)
   ├─ 반복 사용 방지 (다양성)
   └─ 유효한 코스 최대 5개 수집
   
Step 6: 반환
   └─ 5개 중 랜덤 3개 선택하여 반환
```

## 📞 문제 해결

### Q: 응답이 느리면?
```
A: 다음을 확인하세요:
   1. 서버 로그 확인
   2. 네트워크 상태 (카카오맵, 관광공사 API)
   3. 서버 메모리 (통상 <100MB)
```

### Q: 코스 생성이 실패하면?
```
A: 다음을 시도하세요:
   1. 활동 유형 변경
   2. 거리 범위 변경
   3. 다른 위치에서 시도
   4. 서버 로그 확인
```

### Q: 카테고리 분류가 잘못되면?
```
A: 다음을 수정하세요:
   1. classifyPlace() 함수의 정규식 개선
   2. 패턴 우선순위 조정
   3. 새로운 패턴 추가
   4. 테스트 후 재배포
```

### Q: Groq 키가 아직 필요한가?
```
A: 아니요! 완전히 제거됨:
   - server.js에서 모든 Groq 코드 삭제
   - GROQ_API_KEYS 제거
   - getGroqKey() 함수 제거
   - 프롬프트 엔지니어링 불필요
```
